# -План проекта: «Приложение для ведения дневника»
1. Введение

Название проекта:
«Приложение для ведения дневника».

Краткое описание:
Веб-приложение позволяет пользователям вести личный дневник прямо в браузере. Пользователь может создавать, редактировать и удалять записи, просматривать их по датам и искать нужные заметки. Данные сохраняются локально в браузере.

Цели проекта:

•	Сделать удобное и простое приложение для ведения записей.
•	Реализовать хранение данных в браузере (LocalStorage).
•	Добавить возможность поиска по ключевым словам.
•	Обеспечить приятный интерфейс с адаптацией под ПК и мобильные устройства.

Задачи проекта:

•	Реализовать интерфейс с помощью HTML и CSS.
•	Добавить функционал создания и редактирования записей.
•	Настроить удаление и просмотр записей.
•	Реализовать поиск по ключевым словам.
•	Сохранение записей в LocalStorage.

Целевая аудитория:
Пользователи, которые хотят вести дневник без регистрации и сложных настроек — студенты, школьники, все, кому удобно записывать мысли в браузере.

2. Общая информация о проекте

Общий функционал:

•	Создание записей с датой и текстом.
•	Редактирование и удаление записей.
•	Хранение записей в LocalStorage.
•	Просмотр списка записей по датам.
•	Поиск по тексту.
•	Адаптивный интерфейс.

Роли пользователей:

•	Пользователь (единственная роль) — ведёт дневник, управляет записями.

3. Требования к системе

Функциональные требования:

•	Создание и редактирование текстовых записей.
•	Удаление записей.
•	Хранение данных в LocalStorage.
•	Поиск записей по ключевым словам.
•	Просмотр записей в списке.
•	Нефункциональные требования:
•	Интерфейс должен быть простым и удобным.
•	Приложение должно работать офлайн (данные не зависят от сервера).
•	Адаптивная вёрстка (ПК, планшет, телефон).

4. План разработки (4 недели)

Неделя 1. Подготовка проекта

•	Создание структуры проекта (HTML, CSS, JS).
•	Верстка главной страницы.
•	Настройка стилей (CSS, адаптивность).

Неделя 2. Основной функционал

•	Реализация добавления записей.
•	Отображение списка записей.
•	Сохранение данных в LocalStorage.

Неделя 3. Дополнительный функционал

•	Редактирование и удаление записей.
•	Реализация поиска по ключевым словам.
•	Добавление календаря/даты для записей.

Неделя 4. Завершение

•	Тестирование функционала.
•	Рефакторинг кода и улучшение дизайна.
•	Подготовка документации и презентации проекта.
