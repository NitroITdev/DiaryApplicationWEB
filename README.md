# Проект: «DiaryApp - Веб-приложение для ведения дневника»
# Chsherbina Nikita and Kamilzhanov Jahongir

1. Введение

Название проекта:
«DiaryApp - современное веб-приложение для ведения дневника».

Краткое описание:
DiaryApp представляет собой полноценное веб-приложение с клиент-серверной архитектурой, позволяющее пользователям вести личный дневник с безопасной регистрацией и авторизацией. Пользователи могут создавать, редактировать и удалять записи, организовывать их по тегам, искать по содержимому и получать доступ к своим данным с любого устройства.

Цели проекта:

•	Реализовать надежную систему аутентификации с JWT-токенами и верификацией по email.
•	Создать архитектуру клиент-сервер с RESTful API на Go и PostgreSQL.
•	Обеспечить современный и интуитивно понятный пользовательский интерфейс на React.
•	Реализовать полный CRUD для заметок с возможностями поиска и категоризации.

Задачи проекта:

•	Разработать серверную часть на Go с PostgreSQL и RESTful API.
•	Создать клиентскую часть на React с современным UI.
•	Настроить JWT-аутентификацию и валидацию данных.
•	Интегрировать систему верификации через email с Resend API.
•	Реализовать систему тегов и поиска по заметкам.

Целевая аудитория:
Студенты и школьники, ведущие учебные записи; профессионалы, документирующие рабочие процессы; люди, ведущие личный дневник; пользователи, ценящие безопасность и приватность данных.

2. Общая информация о проекте

Общий функционал:

•	Регистрация и авторизация пользователей с email верификацией.
•	Создание, редактирование и удаление заметок с тегами.
•	Полнотекстовый поиск по заголовкам и содержимому заметок.
•	Фильтрация заметок по категориям (Work, Personal, Ideas, Reminders).
•	Безопасное хранение данных в PostgreSQL с JWT аутентификацией.
•	Адаптивный интерфейс для всех устройств.

Роли пользователей:

•	Зарегистрированный пользователь — создает аккаунт, подтверждает email, управляет своими заметками.

Технологический стек:

•	Backend: Go + Gorilla Mux + PostgreSQL + JWT + bcrypt
•	Frontend: React 18 + React Router + Font Awesome
•	Email: Resend API для верификации
•	База данных: PostgreSQL с таблицами users и notes

3. Требования к системе

Функциональные требования:

•	Система регистрации с валидацией паролей и email верификацией.
•	JWT аутентификация с токенами на 24 часа.
•	CRUD операции для заметок через RESTful API.
•	Поиск и фильтрация заметок в реальном времени.
•	Категоризация заметок по тегам.

Нефункциональные требования:

•	Безопасное хеширование паролей с bcrypt.
•	Защита от SQL инъекций и XSS атак.
•	CORS настройка для безопасности.
•	Адаптивная верстка для всех устройств.
•	Время ответа API не более 200ms.

4. Архитектура и развертывание

Backend структура:

•	main.go - точка входа сервера на порту 8080
•	handlers/ - HTTP обработчики для auth и notes
•	middleware/ - JWT авторизация
•	models/ - структуры данных User и Note
•	utils/ - JWT, валидация, email утилиты

Frontend структура:

•	src/pages/ - компоненты страниц (Auth, Diary, Verification)
•	src/api.jsx - HTTP клиент для работы с API
•	src/styles/ - CSS стили с Bootstrap Grid

Развертывание:

•	Backend: localhost:8080 (Go сервер)
•	Frontend: localhost:3000 (React dev server)
•	Database: PostgreSQL с базой diarydb
•	Email: Resend API для отправки кодов верификации
